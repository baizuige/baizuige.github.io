作用域与闭包

2021-01-25 笔记 1k 字
闭包 JavaScript
  • 本文中的示例将使用let和const进行变量和常量的声明。
  • 对于var这种过去式，个人不推荐使用。
  • 此外，示例代码默认是在严格模式下执行。

在解释闭包前，先要了解几个概念：什么是作用域，什么是嵌套函数，什么是垃圾回收。

作用域

作用域就是当前代码的执行上下文。可以在全局作用域下创建多个子作用域，子作用域中的变量及函数无法在外部作用域被访问。

在ES6之前，JavaScript由于var的特性，表面上没有块级作用域（{...}）的相关功能，但JavaScript有着完好的函数作用域（即：在函数中的变量等属性无法被外界访问），因此可以用立即执行函数（(function(){...})()）来模拟块级作用域。

ES6引入了新的let和const关键字来声明变量和常量。let关键字可以将变量绑定到所在的任意作用域中（通常是{ .. }内部）。换句话说，let为其声明的变量隐式地劫持了所在的块作用域。这也就使JavaScript真正有了块作用域。

{
	let message = "HelloWorld!";
	console.log(message); //HelloWorld!
}
console.log(message); //ReferenceError: message is not defined

嵌套函数

当一个函数是在另一个函数中创建的时，那么该函数就被称为“嵌套”的。在JavaScript中很容易实现这一点。

function sayHiBye(firstName, lastName) {
    function getFullName() {
        return firstName + " " + lastName;
    }
    console.log( "Hello, " + getFullName() ); //Hello, bai ge
    console.log( "Bye, " + getFullName() ); //Bye, bai ge
}
sayHiBye("bai","ge");

此外，还可以将嵌套函数作为返回值赋值给外部变量。

function func(){
    return function(){
        console.log("HelloWorld");
    }
}
let hi = func();
hi(); //HelloWorld

垃圾回收

JavaScript会通过自动内存管理实现内存分配和闲置资源回收。基本思路很简单：确定哪个变量不会再使用，然后释放它占有的内存。

通常来说，函数中的局部变量会在函数执行时存在。函数内部使用了变量，然后退出，此时就不再需要这个局部变量了，它占用的内存可以释放。

闭包

闭包是指内部函数总是可以访问其所在的外部函数中声明的变量和参数，即使在其外部函数被返回（寿命终结）了之后。在 JavaScript 中，所有函数都是天生闭包的。

JavaScript 中的函数会自动通过隐藏的 [[Environment]] 属性记住创建它们的位置，所以它们都可以访问外部变量。从技术上讲：所有函数都有名为 [[Environment]] 的隐藏属性，该属性保存了对创建该函数的词法环境的引用。

function foo(){
    let a = 2;
    function bar(){
        console.log(a);
    }
    return bar;
}
let baz = foo();
baz(); //2 这就是闭包的效果

函数bar()的词法作用域能够访问foo()的内部作用域。然后将bar()函数本身当作一个值类型进行传递。在foo()执行后，其返回值（也就是内部的bar()函数）赋值给变量baz并调用baz()，实际上只是通过不同的标识符引用调用了内部的函数bar()。在这个例子中，它在自己定义的词法作用域以外的地方执行。

在foo()执行后，通常会期待foo()的整个内部作用域都被销毁。但是闭包可以阻止这件事情的发生。由于bar()所声明的位置，它拥有涵盖foo()内部作用域的闭包，使得该作用域能够一直存活，以供bar()在之后任何时间进行引用。bar()依然持有对该作用域的引用，而这个引用就叫作闭包。

因此，在几微秒之后变量baz被实际调用（调用内部函数bar），不出意外它可以访问定义时的词法作用域，因此它也可以如预期般访问变量a。这个函数在定义时的词法作用域以外的地方被调用。闭包使得函数可以继续访问定义时的词法作用域。
